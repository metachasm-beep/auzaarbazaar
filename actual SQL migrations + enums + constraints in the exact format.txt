1) Supabase Storage bucket layout

Recommended buckets

Use two buckets (keeps public pages fast, while sensitive files stay private):

public-assets (public)For org logos + listing thumbnails (safe to be public)

private-files (private)For listing original images, PDFs/spec sheets, and message attachments

Folder conventions

public-assets (public)

orgs/{orgId}/logo.{ext}

orgs/{orgId}/banner.{ext}



listings/{listingId}/thumb_{w}x{h}.{ext}

listings/{listingId}/cover_thumb.{ext}	



private-files (private)

listings/{listingId}/images/{uuid}.{ext}

listings/{listingId}/docs/{uuid}-{filename}.pdf



messages/{threadId}/{uuid}-{filename}.{ext}



orgs/{orgId}/kyc/{uuid}-{filename}.pdf



DB rule: store only the object path (e.g. listings/<listingId>/images/<uuid>.jpg) in DB, never store signed URLs.

2) Signed URL strategy (Next.js)

What to do

Public pages show public thumbnails from public-assets directly.

For anything in private-files, generate signed URLs server-side.

TTL recommendations

Listing image galleries: 1 hour signed URLs

PDFs/spec docs: 10–30 minutes

Message attachments: 5–15 minutes (more sensitive)

Server-side signing (Route Handler)

// app/api/storage/sign/route.ts

import { NextResponse } from "next/server";

import { createServerClient } from "@supabase/ssr";

import { cookies } from "next/headers";



export async function POST(req: Request) {

  const { bucket, paths, expiresIn = 3600 } = await req.json();



  const cookieStore = await cookies();

  const supabase = createServerClient(

    process.env.NEXT_PUBLIC_SUPABASE_URL!,

    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,

    { cookies: { get: (n) => cookieStore.get(n)?.value } }

  );



  const { data: { user } } = await supabase.auth.getUser();

  if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });



  // Option A (simple): one-by-one createSignedUrl

  const signed = await Promise.all(

    (paths as string[]).map(async (path) => {

      const { data, error } = await supabase.storage

        .from(bucket)

        .createSignedUrl(path, expiresIn);

      if (error) return { path, error: error.message };

      return { path, url: data.signedUrl };

    })

  );



  return NextResponse.json({ signed });

}



Caching tip (important)

Signed URLs shouldn’t be cached publicly. Use:

Cache-Control: private, max-age=60 on the signing endpoint response

In UI, cache in memory (React state) for the page session

5) Prisma client usage patterns (secure with RLS)

The key point

If you want Postgres RLS to work with Prisma, you must set the Supabase JWT claims on the DB connection before running queries.

Best pattern: wrap every request in a Prisma $transaction and do set_config(...) first, then run queries. That keeps the same connection for the transaction.

5.1 Create a “RLS Prisma” helper

// lib/prisma-rls.ts

import { PrismaClient } from "@prisma/client";



const prisma = new PrismaClient();



type SupabaseClaims = {

  sub: string;           // user id (uuid)

  role: "authenticated"; // important

  email?: string;

};



export async function withRls<T>(

  claims: SupabaseClaims,

  fn: (tx: PrismaClient) => Promise<T>

) {

  const claimsJson = JSON.stringify(claims);



  return prisma.$transaction(async (tx) => {

    // Set the claims for auth.uid(), auth.role(), etc.

    await tx.$executeRawUnsafe(

      `select set_config('request.jwt.claims', '${claimsJson.replaceAll("'", "''")}', true);`

    );

    await tx.$executeRawUnsafe(

      `select set_config('request.jwt.claim.role', 'authenticated', true);`

    );



    return fn(tx as any);

  });

}



5.2 Use it in a Next.js Server Action / Route

// app/api/buyer/inquiries/route.ts

import { NextResponse } from "next/server";

import { createServerClient } from "@supabase/ssr";

import { cookies } from "next/headers";

import { withRls } from "@/lib/prisma-rls";



export async function GET() {

  const cookieStore = await cookies();

  const supabase = createServerClient(

    process.env.NEXT_PUBLIC_SUPABASE_URL!,

    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,

    { cookies: { get: (n) => cookieStore.get(n)?.value } }

  );



  const { data: { user } } = await supabase.auth.getUser();

  if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });



  const data = await withRls(

    { sub: user.id, role: "authenticated", email: user.email ?? undefined },

    async (tx) => {

      return tx.inquiry.findMany({

        orderBy: { createdAt: "desc" },

        include: { listing: true },

      });

    }

  );



  return NextResponse.json({ data });
