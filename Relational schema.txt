Relational schema (Postgres-friendly) to support the Buyer + Seller dashboards



I’m keeping it normalized where it matters, and using JSONB only where specs differ by category.

Core principles

One org can have many users (seller teams, buyer procurement teams).

A user can belong to multiple orgs with roles (owner/admin/member).

Buyers and sellers are both “orgs” with different org_type.

Listings belong to seller orgs.

RFQs belong to buyer orgs and can be broadcast to many seller orgs.

Chat is per “thread” anchored to either an RFQ or a Listing inquiry.

Verification is per seller org (and optionally per listing).

Notes on filtering specs (important)

You’ll have two levels:

Fixed columns in listings for common filters (condition, location, warranty, lead_time, price).

Category-specific filters via category_attributes + listing_specs.

This gives you:

Fast queries for common filters

Flexible specs without schema churn
